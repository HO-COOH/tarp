Detailed Description

The abstract class of language-specific code generators.

To create a code generator of a specific language, any subclasses needs to

  1. inherit this class
  2. call ProjectGenerator::registerCodeGenerator("<language name>", FactoryFunction); where the FactoryFunction accepts a const reference to a ProjectDirCreator object and returns a std::unique_ptr of this class, ie. to have this signature: std::unique_ptr<CodeGenerator> FactoryFunction(ProjectDirCreator const&); For example, a JavaGenerator
    class JavaGenerator: public CodeGenerator
    {
    private:
    bool registered = ProjectGenerator::registerCodeGenerator("java",
    [](ProjectDirCreator const& dir) -> std::unique_ptr<CodeGenerator>
    {
    return std::make_unique<JavaGenerator>(dir);
    });
    public:
    //...
    };
  3. Provide the definition of these 2 virtual functions:
    void generate();
    [[nodiscard]] std::string_view getGeneratedFilename() const;
    Also define a constructor that accepts a ProjectDirCreator as a const reference, which provides the necessary location for the code generator to write the generated code:
    explicit JavaGenerator(ProjectDirCreator const& projectDir) : CodeGenerator(projectDir) {}

Q: Why use inheritance instead of reading a configuration file? A: Because I thought some language-specific code generators may take more steps, like checking the project type, checking for system information etc... Therefore using inheritance provided greater flexibility, while still allowing a modular design.
